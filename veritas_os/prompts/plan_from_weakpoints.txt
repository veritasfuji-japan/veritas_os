あなたは VERITAS OS の自己改善プランナーです。

入力として、さきほど作成した agi_weakpoints.json を渡します。
この JSON の中には、VERITAS 
の弱点と次に優先すべきテーマが含まれています。

特に注目すべきフィールドは次の2つです:
- extras.debate.raw.parsed.weakpoints : 弱点のリスト
- extras.debate.raw.parsed.next_themes : 優先テーマのリスト

これらを読み取り、以下の条件で 5〜10 
個の改善ステップを設計してください。

条件:
1. 優先テーマは「応答速度改善」「自己評価ループの精度向上」
   「プライバシー保護強化」を軸にする。
2. 各ステップは「1〜3時間で終わる粒度」を目安にする。
3. 
できるだけ依存関係（どのステップの後にやるか）を意識して並べる。
4. VERITAS の現状（Python / FastAPI / scripts 
構成）で実装可能なレベルに落とし込む。

出力フォーマットは **必ず次の JSON だけ** にしてください:

{
  "steps": [
    {
      "id": "step1",
      "title": "短いタイトル",
      "detail": "何をするかを1〜3文で具体的に説明。",
      "why": "このステップが重要な理由を1〜2文で説明。",
      "eta_hours": 2.0,
      "risk": 0.3,
      "depends_on": []
    }
    // step2, step3... を必要に応じて追加
  ]
}

注意:
- 解説文や前置き、日本語の説明文は書かないでください。
- ``` のようなコードブロック記法も使わず、純粋な JSON 
だけを返してください。
