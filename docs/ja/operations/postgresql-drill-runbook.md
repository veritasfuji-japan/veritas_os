# PostgreSQLドリルRunbook（日本語解説）

## 位置づけ
障害復旧ドリルを定期運用するための日本語解説です。運用チームが手順の意図を理解するための入口として利用します。

## 要点
- ドリルは「手順がある」ことではなく「実行して再現できる」ことを証明します。
- バックアップ、復旧、整合性確認、証跡記録を一連で実施します。
- 失敗時は fail-closed の安全側停止手順を優先します。

## VERITASにおける意味
- Replay / 再現実行 と運用ドリルの両方が揃うことで、監査説明力が上がります。
- Mission Control と TrustLog の参照可能性を継続確認できます。

## 実装上の確認ポイント
- ドリル実行ログ、所要時間、失敗ケースを記録する。
- 復旧後に bind証跡参照APIが正常か確認する。
- 詳細は英語正本または実装ファイルを確認してください。

## 現時点の制限
- ドリル結果は対象環境に限定されます。
- 全障害モードを網羅した保証ではありません。
