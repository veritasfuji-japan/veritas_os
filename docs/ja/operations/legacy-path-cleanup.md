# レガシーパスクリーンアップ（日本語解説）

## 位置づけ
旧経路と新経路の混在による運用リスクを減らすための整理文書です。

## 要点
- 互換維持と削除計画を分離し、段階的に移行します。
- 旧経路を残す場合は監査・運用への影響を明示します。
- README と docs index のリンク整合を維持します。

## VERITASにおける意味
- operator-facing governance surface の一貫性を保つため、旧API/旧導線の整理が必要です。
- Replay と証跡参照の再現性維持にも寄与します。

## 実装上の確認ポイント
- 旧経路の deprecate 状態と削除予定を台帳化する。
- 参照先更新後に docs リンク解決チェックを実行する。
- 詳細は英語正本または実装ファイルを確認してください。

## 現時点の制限
- 互換維持期間中は重複実装コストが残ります。
- 本番変更時は段階リリースとロールバック計画が必要です。
