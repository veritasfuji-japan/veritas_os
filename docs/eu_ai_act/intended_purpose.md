## 1. 意図された用途

### 1.1 主要用途

VERITAS OSは以下の用途を意図して設計されている:

| 用途 | 説明 | リスク分類 |
|------|------|-----------|
| **企業AI意思決定支援** | ビジネス上の意思決定に対するAI支援分析 | 用途依存 |
| **AIガバナンス管理** | AI利用の安全性・コンプライアンス監視 | 限定リスク |
| **規制環境下でのLLM安全運用** | 安全ゲート付きLLM推論 | 用途依存 |

### 1.2 対象ユーザー

| ユーザー種別 | 前提条件 |
|------------|---------|
| **開発者 / システム管理者** | AIシステム運用の技術知識を有する |
| **ビジネスアナリスト** | 意思決定プロセスの理解を有する |
| **コンプライアンス担当者** | 規制要件の理解を有する |
| **人間レビュアー** | 高リスク決定の承認権限を有する |

### 1.3 対象地域・言語

- **主要対応言語**: 英語、日本語
- **追加対応**: Art. 5禁止慣行検出はフランス語・ドイツ語・スペイン語にも対応
- **展開地域**: 全世界（ただしEU域内展開時はEU AI法の完全遵守が必要）

## 2. 禁止用途

以下の用途でのVERITAS OS使用は**明示的に禁止**する（EU AI法第5条準拠）:

### 2.1 絶対禁止

| 禁止用途 | 根拠条文 |
|---------|---------|
| サブリミナル技術による行動操作 | Art. 5(1)(a) |
| 社会的スコアリングシステム | Art. 5(1)(c) |
| リアルタイム遠隔バイオメトリクス識別 | Art. 5(1)(h) |
| 脆弱性の悪用（年齢・障害等） | Art. 5(1)(b) |
| 予測的取締（犯罪予測警察活動） | Art. 5(1)(d) |
| 感情認識（職場・教育） | Art. 5(1)(f) |
| 無差別な顔画像スクレイピング | Art. 5(1)(e) |

### 2.2 条件付き禁止（人間監視なしでの使用禁止）

| 用途 | 条件 |
|------|------|
| 雇用・採用意思決定 | 人間レビュー必須 (Art. 14) |
| 信用評価 | 人間レビュー必須 (Art. 14) |
| 教育評価 | 人間レビュー必須 (Art. 14) |
| 医療補助判断 | 人間レビュー必須 (Art. 14) + 医療専門家監督 |
| 司法・法執行支援 | 人間レビュー必須 (Art. 14) + 法務レビュー |

## 3. システムの能力と限界

### 3.1 能力

| 能力 | 説明 |
|------|------|
| **多角的分析** | 複数の観点からの意思決定分析（debate機能） |
| **リスク評価** | Annex III高リスクドメインの自動検出・スコアリング |
| **説明可能性** | 推論トレース・証拠・批評の提供 |
| **安全ゲート** | 4層の安全ゲート（FUJI Gate）によるリスク制御 |
| **監査追跡** | SHA-256ハッシュチェーン付き改ざん防止ログ |
| **人間監視** | 高リスク決定の自動一時停止と人間レビュー要求 |

### 3.2 既知の制限

| 制限 | 影響 | 緩和策 |
|------|------|--------|
| **LLM幻覚（ハルシネーション）** | 事実と異なる情報を生成する可能性 | 証拠ソースの明示、不確実性スコア提供 |
| **非決定性** | 同一入力に対する出力が毎回異なる可能性 | 不確実性スコアによる信頼度表示 |
| **領域外質問** | 訓練データ範囲外の専門分野への対応が不十分 | リスクスコアによる警告 |
| **言語依存性** | 英語・日本語以外の精度が低下する可能性 | 対応言語の明示 |
| **リアルタイム情報の欠如** | LLMの訓練データカットオフ以降の情報を反映しない | 外部データソース統合の検討 |
| **第三者モデル依存** | OpenAI APIの可用性に依存 | 縮退モードの検討（P3対応） |

### 3.3 不確実性と信頼度

VERITAS OSは各決定に対して以下の信頼度指標を提供する:

- **uncertainty**: 不確実性スコア（0.0 = 完全確信 〜 1.0 = 完全不確実）
- **telos_score**: 価値整合スコア
- **risk_score**: Annex IIIリスクスコア

**重要**: これらのスコアはヒューリスティックに基づくものであり、法的判断の唯一の根拠として使用してはならない。

## 4. 人間監視の要件

### 4.1 監視レベル

| リスクレベル | 要求される監視 |
|------------|--------------|
| **LOW** (risk_score < 0.5) | 監視不要（自動処理） |
| **MEDIUM** (0.5 ≤ risk_score < 0.85) | 定期的な監視推奨 |
| **HIGH** (risk_score ≥ 0.85) | 人間レビュー必須（`PENDING_HUMAN_REVIEW`） |

### 4.2 人間レビュープロセス

1. 高リスク決定が検出されると`PENDING_HUMAN_REVIEW`ステータスが付与される
2. レビュアーへWebhook通知が送信される（設定済みの場合）
3. SLAデッドライン（デフォルト4時間）内にレビューが必要
4. レビュアーは承認 (`approved`) または拒否 (`rejected`) を決定する
5. `fail_close=True`（デフォルト）の場合、人間承認なしでは決定は実行されない

## 5. 異議申し立て

### 5.1 異議申し立て権

AIシステムによる決定の影響を受ける者は、以下の方法で異議を申し立てることができる:

1. **内部異議申し立て**: システム管理者に対する決定の再検討要求
2. **人間レビュー要求**: `human_review`アクションの手動トリガー
3. **決定の取消要求**: 監査ログ（TrustLog）に基づく決定の撤回要求
