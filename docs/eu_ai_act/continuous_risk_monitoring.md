## 1. 目的

本手順書は、EU AI Act 第9条が要求するライフサイクル全体にわたる継続的リスク管理システムの運用手順を定める。VERITAS OS の開発・テスト・デプロイ・運用・廃止の各フェーズにおけるリスクモニタリング活動を体系化し、準拠状態を維持する。

---

## 2. スコープ

- VERITAS OS 本体（Decision Pipeline、FUJI Gate、TrustLog、MemoryOS）
- 使用する第三者AIモデル（OpenAI GPT-4.1-mini 等）
- 高リスク展開シナリオ（雇用支援・医療補助・金融判定 等）

## 4. エスカレーションフロー

### 4.1 インシデント分類

| レベル | 説明 | 対応期限 | エスカレーション先 |
|--------|------|---------|-------------------|
| **Critical** | 禁止慣行（Art.5）検出、データ漏洩、人権侵害 | 即時 | 経営層 + 法務 + 規制当局 |
| **High** | 精度大幅低下、バイアス検出、SLA 違反 | 4時間以内 | リスク管理者 + 開発リーダー |
| **Medium** | 軽微な精度低下、設定不備 | 24時間以内 | 運用チームリーダー |
| **Low** | ドキュメント不備、軽微な改善点 | 1週間以内 | 担当者 |

### 4.2 対応プロセス

1. **検知（Detection）**: 自動監視ツールまたは手動レビューによる検知
2. **評価（Evaluation）**: インシデントレベルの分類と影響範囲の特定
3. **対応（Response）**: 是正措置の実施（必要に応じてシステム停止）
4. **報告（Reporting）**: ステークホルダーへの報告と記録
5. **根本原因分析（Root Cause Analysis）**: 再発防止策の策定

## 5. KPI と閾値

| 指標 | 目標値 | 警告閾値 | 危険閾値 |
|------|--------|---------|---------|
| TrustLog チェーン整合性 | 100% | < 99.9% | < 99% |
| 精度ドリフト（全体平均との差） | < 2% | 2-5% | > 5% |
| 人間レビュー SLA 遵守率 | ≥ 95% | 90-95% | < 90% |
| 偽陰性率（Art.5 検出） | < 1% | 1-5% | > 5% |
| バイアス群間差分 | < 5% | 5-10% | > 10% |
| インシデント平均対応時間 | < 2時間 | 2-4時間 | > 4時間 |

## 6. ツールとインフラ

### 6.1 自動化ツール

| ツール | 用途 | 実行頻度 |
|--------|------|---------|
| `scripts/doctor.py` | TrustLog 検証 + 精度ベンチマーク | 日次 CI |
| `eu_ai_act_compliance_module.py` | リスク分類 + 人間レビューキュー | リアルタイム |
| `compliance/report_engine.py` | コンプライアンスレポート生成 | 月次 |
| `assess_continuous_risk_monitoring()` | モニタリング完了状況の評価 | 月次 |

### 6.2 プログラム的利用

```python
from veritas_os.core.eu_ai_act_compliance_module import (
    assess_continuous_risk_monitoring,
    RISK_MONITORING_SCHEDULE,
)

# モニタリング活動の完了状況を評価
result = assess_continuous_risk_monitoring(
    completed_activities={
        "daily": ["trust_log_integrity_check", "anomaly_detection_review"],
        "weekly": ["accuracy_drift_analysis"],
        "monthly": ["risk_register_update"],
    }
)

print(f"Overall compliance score: {result['overall_score']}")
print(f"Compliant: {result['compliant']}")
```
