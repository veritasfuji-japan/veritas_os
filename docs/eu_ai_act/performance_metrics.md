## 2. 精度指標の定義

### 2.1 主要精度指標（KPI）

| 指標 | 定義 | 目標値 | 測定方法 |
|------|------|--------|---------|
| **意思決定精度** | 推奨選択肢が専門家判断と一致する割合 | ≥ 0.80 | ベンチマークテスト |
| **Art. 5検出率（再現率）** | 禁止慣行を含むテキストの検出率 | ≥ 0.95 | red-teamテスト |
| **Art. 5誤検出率（偽陽性率）** | 正常テキストを禁止慣行として誤判定する割合 | ≤ 0.05 | ベンチマークテスト |
| **リスク分類精度** | Annex IIIリスクレベルの正確な判定割合 | ≥ 0.85 | ラベル付きテストデータ |
| **PII検出再現率** | 個人情報を含むテキストの検出率 | ≥ 0.90 | 合成PIIテストデータ |
| **プロンプトインジェクション検出率** | 悪意あるプロンプトの検出率 | ≥ 0.85 | 攻撃パターンテスト |

### 2.2 堅牢性指標

| 指標 | 定義 | 目標値 | 測定方法 |
|------|------|--------|---------|
| **入力擾乱耐性** | ノイズ入力に対する出力安定性 | > 0.90相関 | 擾乱テスト |
| **多言語一貫性** | 同一意味の多言語入力に対する判定一致率 | ≥ 0.80 | 翻訳対テスト |
| **負荷耐性** | 高負荷時の応答精度維持率 | ≥ 0.95 | 負荷テスト |
| **回避攻撃耐性** | 回避テクニックに対する検出維持率 | ≥ 0.85 | red-teamテスト |

## 4. 継続的精度モニタリング

### 4.1 モニタリング基盤

VERITAS Doctor (`veritas_os/scripts/doctor.py`) が以下の継続的モニタリングを提供する:

| 機能 | 説明 | 出力 |
|------|------|------|
| **精度集計** | ベンチマーク結果の統計分析（平均・最小・最大） | `doctor_report.json` → `accuracy` セクション |
| **ドリフト検出** | 直近5回の精度が全体平均から5%以上低下した場合にアラート | `drift_detected: true` |
| **TrustLog整合性** | ハッシュチェーン検証 | `trustlog` セクション |

### 4.2 ドリフト検出アルゴリズム

```python
# doctor.py: analyze_accuracy_benchmarks()
if len(accuracies) >= 5:
    overall_avg = statistics.mean(accuracies)
    recent_avg = statistics.mean(accuracies[-5:])
    drop = overall_avg - recent_avg
    if drop > 0.05:
        drift_detected = True
```

### 4.3 ベンチマーク実行手順

`AGI_BENCH_INTEGRATION_GUIDE.md` に基づく:

1. テストデータ準備（合成データ使用必須）
2. ベンチマーク実行
3. 結果を `benchmarks/results/` に JSON形式で保存
4. `doctor.py` で精度集計・ドリフト検出を実行
5. 結果を本文書 §3.2 / §3.3 に反映

### 4.4 結果ファイル形式

ベンチマーク結果は以下のJSON形式で `benchmarks/results/` に保存する:

```json
{
  "benchmark": "ベンチマーク名",
  "model": "gpt-4.1-mini",
  "timestamp": "2026-03-06T12:00:00Z",
  "accuracy": 0.85,
  "total_cases": 100,
  "passed_cases": 85,
  "details": {}
}
```
