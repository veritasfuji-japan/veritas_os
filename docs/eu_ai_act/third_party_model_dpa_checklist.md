## 5. データフロー

```
ユーザー入力
    ↓
VERITAS OS パイプライン
    ↓ (API呼出)
OpenAI API (GPT-4.1-mini)
    ↓ (API応答)
VERITAS OS 安全ゲート
    ↓
ユーザー出力
    ↓
TrustLog (ローカル保存)
```

### 5.1 OpenAIに送信されるデータ

- ユーザープロンプト（PII マスキング済み — `core/sanitize.py`）
- システムプロンプト（VERITAS OS内部設定）
- コンテキスト情報（匿名化済み）

### 5.2 OpenAIに送信されないデータ

- ユーザー識別情報（user_id はハッシュ化）
- TrustLog（ローカルのみ）
- メモリストレージ（ローカルのみ）

## 6. リスク評価

| リスク | 影響度 | 緩和策 |
|--------|--------|--------|
| API経由のデータ漏洩 | 高 | PII マスキング、TLS暗号化、APIキー管理 |
| 訓練データへの利用 | 中 | OpenAI API Data Usage Policy確認（API利用は訓練に不使用を確認） |
| EU外へのデータ移転 | 高 | SCC/適切な移転メカニズムの確認 |
| サブプロセッサのリスク | 中 | サブプロセッサ一覧の定期確認 |
| モデル更新によるリグレッション | 中 | バージョン固定・更新時再評価 |
