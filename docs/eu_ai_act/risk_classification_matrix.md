## 1. 目的

本文書はEU AI法（Regulation (EU) 2024/1689）第6条および附属書Ⅲに基づき、VERITAS OSが展開される可能性のあるユースケース別にリスク分類を定義する。

各展開シナリオにおける適用条文と遵守義務を明示し、プロバイダー・デプロイヤーが法的義務を把握できるようにする。

## 3. リスク分類マトリクス

### 3.1 分類基準

EU AI法は以下の4段階でAIシステムを分類する：

| リスクレベル | 条文 | 概要 |
|-------------|------|------|
| **許容不可能（Unacceptable）** | 第5条 | 全面禁止。サブリミナル操作、社会スコアリング等 |
| **高リスク（High-Risk）** | 第6条・附属書Ⅲ | 厳格な義務（第9〜15条の完全遵守） |
| **限定リスク（Limited Risk）** | 第50条 | 透明性義務のみ |
| **最小リスク（Minimal Risk）** | — | 自主規制のみ |

### 3.2 展開シナリオ別分類

| # | 展開シナリオ | リスク分類 | 附属書Ⅲ該当 | 適用条文 | 遵守義務 |
|---|-------------|-----------|-------------|---------|---------|

## 5. VERITAS OS自体のリスク分類宣言

### 5.1 基本分類

VERITAS OS自体は**汎用意思決定支援システム**であり、単独では高リスクAIシステムに分類されない。ただし、**展開用途**によって高リスクに分類される可能性がある。

| 条件 | 分類 |
|------|------|
| 一般業務用途（S-01, S-02） | 限定リスク〜最小リスク |
| 附属書Ⅲ該当用途（S-03〜S-09） | **高リスク** |
| 生体認証用途（S-10） | **許容不可能〜高リスク** |

### 5.2 GPAIとしての分類

VERITAS OSがOpenAI GPT-4.1-miniを使用している場合、EU AI法第51-56条の**汎用AIモデル（GPAI）**に関する規定が適用される可能性がある。GPAI提供者としての義務は主にOpenAIが負うが、VERITAS OSのデプロイヤーはダウンストリーム責任を負う。

高リスクシナリオ（S-03〜S-09）でVERITAS OSを展開する前に、以下を確認すること：

- [ ] リスク分類の正式決定（本マトリクスに基づく）

- [ ] モデルカード・バイアス評価の完了
- [ ] DPA（データ処理契約）の締結確認
- [ ] 人間レビューワークフローの運用テスト
- [ ] ログ保持期間 ≥ 6ヶ月の確認
- [ ] AI開示フィールドの全出力への付与確認
- [ ] 法務レビュー・DPOレビューの完了
- [ ] 適合性評価の計画策定
- [ ] CEマーキングプロセスの準備（該当する場合）
