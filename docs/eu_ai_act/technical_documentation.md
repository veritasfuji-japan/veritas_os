## 1. システム概要

### 1.1 システム識別情報

| 項目 | 内容 |
|------|------|

| **開発者/プロバイダー** | [組織名を記入] |
| **連絡先** | [連絡先を記入] |
| **EU代表者** | [EU域内代表者（必要な場合）を記入] |

### 1.2 システムタイプ

VERITAS OSは、大規模言語モデル（LLM: OpenAI GPT-4.1-mini）をラップし、安全ゲート付き意思決定パイプラインを提供する汎用AIシステム（GPAI）基盤の意思決定支援システムである。

### 1.3 アーキテクチャ概要

```
ユーザー入力 → FUJI Gate（安全・リスク評価）→ LLM推論 → 出力検証 → TrustLog記録
                  ↓                                         ↓
           リスク分類 (Art.9)                        Art.5禁止慣行チェック
           PII検出 (Art.15)                         人間監視フック (Art.14)
           プロンプトインジェクション検出               AI開示付与 (Art.50)
```

### 2.2 品質保証プロセス

- **静的解析**: ruff (Python lint + format)

主要依存関係はSBOM (`security/sbom/`) に完全記載。以下は主要コンポーネント:

| コンポーネント | 技術 | 用途 |
|-------------|------|------|
| 推論エンジン | OpenAI GPT-4.1-mini | LLM推論 |
| 安全ゲート | FUJI Gate | リスク評価・フィルタリング |
| ログ基盤 | TrustLog (SHA-256チェーン) | 改ざん防止監査ログ |

## 3. 設計仕様

### 3.1 入出力仕様

**入力**:
- テキスト形式の意思決定クエリ（`/v1/decide` エンドポイント）
- オプション群（選択肢）、コンテキスト情報

**出力**:
- 推奨選択肢 (`chosen`) と不確実性スコア
- 推論トレース (`reasoning_trace`)
- 証拠 (`evidence`) と出所情報
- 批評 (`critique`) と多角的議論 (`debate`)
- リスク評価 (`fuji`) とセーフティゲート判定
- AI開示ラベル（Art. 50準拠）

### 3.2 データフロー

1. ユーザー入力受信
2. Art. 5入力検査（禁止慣行チェック）
3. Art. 9リスク分類（Annex III高リスクドメイン判定）
4. FUJI Gate安全評価
5. LLM推論実行
6. 出力検証（Art. 5出力検査）
7. 人間監視フック適用（Art. 14）
8. TrustLog記録（Art. 12）
9. AI開示付与（Art. 50）
10. レスポンス返却
