# Security Hardening Checklist

> **Audience**: Operations / DevOps teams deploying Veritas OS to production  
> **Last updated**: 2026-03-28

This document provides a comprehensive security checklist for hardening Veritas OS in production deployments.

---

## 1. API Secret & Authentication

| Item | Action | Priority |
|------|--------|----------|
| Set `VERITAS_API_SECRET` | Generate a strong random secret (≥ 32 chars). **Never** use placeholder values. | **CRITICAL** |
| Set `VERITAS_ENFORCE_API_SECRET=1` | Enforced by default; do not override to `0` in production. | **CRITICAL** |
| Enable external secret manager | Set `VERITAS_ENFORCE_EXTERNAL_SECRET_MANAGER=1` with a supported `VERITAS_SECRET_PROVIDER` (vault, aws_secrets_manager, gcp_secret_manager, azure_key_vault, kms). | HIGH |
| Set `VERITAS_API_SECRET_REF` | Point to the external secret name/path when using an external provider. | HIGH |
| Rotate API secrets periodically | Implement key rotation via your secret manager. | HIGH |
| Restrict SSE/WS query-param API keys | Do **not** set `VERITAS_ALLOW_SSE_QUERY_API_KEY=1` or `VERITAS_ALLOW_WS_QUERY_API_KEY=1` in production. API keys in query strings risk exposure in logs and referrers. | HIGH |

---

## 2. Encryption

| Item | Action | Priority |
|------|--------|----------|
| Set `VERITAS_ENCRYPTION_KEY` | Generate with `python -c "from veritas_os.logging.encryption import generate_key; print(generate_key())"`. Required for TrustLog writes. | **CRITICAL** |
| Store key externally | Keep the encryption key in Vault/KMS, not in `.env` files or source control. | **CRITICAL** |
| Disable legacy decryption | Ensure `VERITAS_ENCRYPTION_LEGACY_DECRYPT` is unset (default) unless actively migrating. | MEDIUM |

---

## 3. TrustLog Signing

| Item | Action | Priority |
|------|--------|----------|
| Protect Ed25519 private key | Store the signing key in an HSM, Vault, or KMS. The current file-based key storage (`trustlog_signed.py`) is acceptable for development only. | HIGH |
| Enable WORM mirror | Set `VERITAS_TRUSTLOG_WORM_MIRROR_PATH` to a write-once storage path for regulatory compliance. | MEDIUM |
| Enable hard-fail on WORM errors | Set `VERITAS_TRUSTLOG_WORM_HARD_FAIL=1` if audit integrity is a regulatory requirement. | MEDIUM |
| Enable transparency log | Set `VERITAS_TRUSTLOG_TRANSPARENCY_LOG_PATH` and `VERITAS_TRUSTLOG_TRANSPARENCY_REQUIRED=1` for blockchain-like anchoring. | LOW |

---

## 4. Runtime Environment

| Item | Action | Priority |
|------|--------|----------|
| Set `VERITAS_ENV=production` | Activates fail-closed auth, strict CSP, security guards, and production logging levels. | **CRITICAL** |
| Set `VERITAS_RUNTIME_NAMESPACE=prod` | Ensures data isolation from dev/test environments. | HIGH |
| Use Redis auth store | Set `VERITAS_AUTH_SECURITY_STORE=redis` and `VERITAS_AUTH_REDIS_URL` for multi-worker setups. In-memory store is single-process only. | HIGH |
| Set `VERITAS_AUTH_STORE_FAILURE_MODE=closed` | Default is `closed`; never set to `open` in production. | **CRITICAL** |
| Restrict CORS origins | Set `VERITAS_CORS_ALLOW_ORIGINS` to specific trusted origins. Never include `*`. | HIGH |
| Configure trusted proxies | Set `VERITAS_TRUSTED_PROXIES` to your reverse proxy IPs only. | MEDIUM |

---

## 5. Network & Transport

| Item | Action | Priority |
|------|--------|----------|
| TLS termination | Deploy behind a reverse proxy (nginx, Caddy, ALB) with TLS 1.2+. Veritas OS does not terminate TLS itself. | **CRITICAL** |
| Restrict listening interfaces | Bind uvicorn to `127.0.0.1` when behind a reverse proxy. | HIGH |
| Set `VERITAS_MAX_REQUEST_BODY_SIZE` | Limit request body size (default is enforced by middleware). | MEDIUM |
| Configure graceful shutdown | Set `VERITAS_SHUTDOWN_DRAIN_SEC` (default: 10s) appropriate for your load profile. | LOW |

---

## 6. Docker & Container Deployment

| Item | Action | Priority |
|------|--------|----------|
| Non-root execution | The Dockerfile uses `appuser` (non-root). Do not override `USER` in production. | **CRITICAL** |
| Resource limits | CPU/memory limits are set in `docker-compose.yml` (`deploy.resources`). Review and adjust for your workload. | HIGH |
| Read-only filesystem | Mount application code as read-only where possible. Runtime directories (`/runtime`) need write access. | MEDIUM |
| No privileged mode | Never run with `--privileged` or `--cap-add=ALL`. | **CRITICAL** |
| Image scanning | Trivy scanning is built into `publish-ghcr.yml` workflow. Verify scans pass before deployment. | HIGH |
| SBOM tracking | Nightly CycloneDX SBOMs are generated by `sbom-nightly.yml`. Review SBOM drift alerts. | MEDIUM |

---

## 7. Governance & Compliance

| Item | Action | Priority |
|------|--------|----------|
| Enable RBAC | Set `VERITAS_GOVERNANCE_ENFORCE_RBAC=1` (default). | HIGH |
| Four-eyes principle | Set `VERITAS_GOVERNANCE_REQUIRE_FOUR_EYES=1` (default) for policy change dual-approval. | HIGH |
| Restrict governance roles | Set `VERITAS_GOVERNANCE_ALLOWED_ROLES` to only authorized personnel. | HIGH |
| Set tenant ID | Set `VERITAS_GOVERNANCE_TENANT_ID` for multi-tenant deployments. | MEDIUM |
| EU AI Act mode | Set `VERITAS_EU_AI_ACT_MODE=1` if subject to EU AI Act requirements. | Situational |
| Human review webhook | Set `VERITAS_HUMAN_REVIEW_WEBHOOK_URL` for escalation notifications. | HIGH |

---

## 8. Logging & Monitoring

| Item | Action | Priority |
|------|--------|----------|
| Encrypted log directory | Set `VERITAS_ENCRYPTED_LOG_ROOT` and `VERITAS_REQUIRE_ENCRYPTED_LOG_DIR=1` for sensitive logs. | HIGH |
| Log rotation | Set `VERITAS_LOG_MAX_LINES` for log file rotation. | MEDIUM |
| Monitor capability manifests | Review `[CapabilityManifest]` log entries to detect disabled features. | LOW |
| Pipeline warnings | Keep `VERITAS_PIPELINE_WARN=1` (default) to detect anomalies. | LOW |

---

## 9. Web Search / SSRF Prevention

| Item | Action | Priority |
|------|--------|----------|
| Host allowlist | Set `VERITAS_WEBSEARCH_HOST_ALLOWLIST` to restrict outbound web search domains. | HIGH |
| Toxicity filter | Keep `VERITAS_WEBSEARCH_ENABLE_TOXICITY_FILTER=1` (default). | MEDIUM |
| Review SSRF protections | `web_search_security.py` blocks private/local IPs, strips IPv6 scope IDs, and canonicalizes hostnames. No action needed unless extending. | Info |

## 10. PostgreSQL Backend

| Item | Action | Priority |
|------|--------|----------|
| Set `VERITAS_DB_SSLMODE=verify-full` | Full certificate chain + hostname verification for database connections. | **CRITICAL** |
| Use external secret manager for `VERITAS_DATABASE_URL` | Database credentials must not be stored in `.env` files, `docker-compose.yml`, or source control in production. | **CRITICAL** |
| Disable `SUPERUSER` for application role | The PostgreSQL role used by VERITAS should have only `INSERT`, `SELECT`, `UPDATE`, `DELETE` on managed tables. | **CRITICAL** |
| Restrict `pg_hba.conf` | Limit database access to application server IPs only. | **CRITICAL** |
| Set `password_encryption = scram-sha-256` | Use SCRAM-SHA-256 over MD5 for password hashing. | HIGH |
| Set `idle_in_transaction_session_timeout` | Kill leaked transactions that the application fails to close (recommend `60s`). | HIGH |
| Set `VERITAS_DB_STATEMENT_TIMEOUT_MS` ≤ `10000` | Prevent runaway queries from holding advisory locks. | HIGH |
| Set `VERITAS_DB_AUTO_MIGRATE=false` | Run migrations as a discrete deployment step, not on application startup. | HIGH |
| Enable `log_min_duration_statement` | Log slow queries (recommend `500` ms) for performance monitoring. | MEDIUM |
| Enable `pg_stat_statements` | Query performance analytics. | MEDIUM |
| Disable unused PostgreSQL extensions | Reduce attack surface. | LOW |

> See [`docs/postgresql-production-guide.md`](postgresql-production-guide.md) for the full PostgreSQL production guide.

## 11. Pre-Deployment Verification

```bash
# Run production validation (requires running services)
make test-production

# Run smoke tests
pytest -m "production or smoke" veritas_os/tests/

# Verify security gates
make security-audit

# Check dependency advisories
pip-audit
pnpm audit

# Verify SBOM baseline
sha256sum -c security/sbom/baseline/python.cdx.sha256
sha256sum -c security/sbom/baseline/node.cdx.sha256
```
